Introduction

As part of our university studies in computer science, we had the opportunity to complete a project designing a companion virus as part of our cybersecurity course, supervised by Mr. Franco and Mr. Bouju. This project had a specific educational goal: to study and understand the propagation of viruses, particularly companion viruses, in a controlled environment. The main objective was to learn how to develop virus propagation in an academic context, rather than to create a real virus intended for malicious purposes.

Project Context

The project consisted of creating a companion virus, where the infected program hides under the name of a legitimate file, thus tricking the user into executing the malicious program. We were provided with a CD-ROM containing images and a fictitious application called MediaPlayer.exe, which was supposed to read images, but in reality, served as a vector to propagate the virus.

The virus had to infect other executable files in the same directory as the MediaPlayer.exe program, all while pretending to be a legitimate file to avoid raising the user's suspicions. The project was carried out in a Linux Ubuntu environment, and the virus propagation focused on this operating system to simulate a real-world scenario in an academic context. This work also served an educational purpose, deepening our knowledge of cybersecurity.

Concept of Companion Virus

A companion virus is a specific type of malware that hides under the name of a legitimate program, usually an executable file, to deceive the user. It differs from traditional viruses by its propagation method: instead of running by itself, it waits for a legitimate file to be executed by the user to spread. The infected program therefore contains the virus but initially launches as if it were the original program, making it difficult to detect.

In essence, the companion virus acts as a "parasite," adopting the name and path of a legitimate executable file. The user thus believes they are running a normal application, while in reality, the virus runs first, before handing control over to the legitimate program to hide its presence and avoid suspicion. Once executed, the virus seeks out other executable files in the target directory and infects them in turn, continuing its propagation. This approach allows the companion virus to effectively hide while spreading among the files.

The steps of infection in a companion virus can be summarized as follows:

  1. Initial Infection: When a user executes the malicious program, it renames the executable files in the directory and replaces them with copies of itself. The legitimate program is then executed.
  2. Subsequent Infections: The infected files become vectors for propagation as soon as they are executed.

Objectives

The main objectives of the project were:

  1. Develop a credible MediaPlayer: Create a functional program capable of reading images in various formats (.jpg, .png, .bmp), which would serve as a vector for the virus execution.
  2. Create utilities: Develop several simple utilities that would be infected by the virus. These utilities must interact with the user and perform simple tasks like generating passwords or performing calculations.
  3. Implement virus propagation: Design the mechanism by which the virus replicates and infects executable files in the directory. This should occur through initial infection and subsequent infections, with each executed program becoming a vector for propagation.

Project Organization

Since we were working exclusively in person at the university, we chose to organize the development of the project in pairs. This approach allowed us to effectively divide tasks while working closely together to develop the virus and utilities.

  • Pierre developed the MediaPlayer.exe program and the MonPG4 utility.
  • Arthur handled the creation of the other utilities (MonPG1, MonPG2, MonPG3, MonPG5).

To facilitate project management, we used Gitlab for version control, allowing us to centralize the code and collaborate smoothly, while ensuring continuous integration of the different parts of the project.

Development of Utilities

The utilities developed fulfilled specific criteria: they had to perform a useful task, display a graphical interface, and interact with the user. Here are the utilities created:

  • MonPG1: A password generator.
  • MonPG2: A basic calculator for simple operations.
  • MonPG3: A calculator for the hypotenuse based on the Pythagorean theorem.
  • MonPG4: A simple timer with a start/stop button and reset functionality.
  • MonPG5: A "guess the price" game where the user must guess a number chosen by the computer.

MediaPlayer and Virus Propagation

The MediaPlayer.exe program was designed to resemble a legitimate image viewer application. Its primary role was to convince the user to execute it. Upon execution, it carried out the following actions:

  1. During the initial infection, the virus renames the existing executables in the target directory with the .old extension and creates a copy of itself under the original file names.
  2. During subsequent infections, each executed program contained a copy of the virus, which continued to propagate the infection to other executable files in the directory.

Virus Propagation Mechanisms

The companion virus uses a simple strategy of renaming and duplicating executable files to hide and propagate itself. During infection, the following steps are carried out:

  1. Renaming the target file with the .old extension.
  2. Creating a copy of the virus under the original file name, allowing the legitimate program to continue running without alerting the user.
  3. Verification to avoid re-infecting already compromised files.

Obfuscation and Security

We studied code obfuscation to make the virus harder to analyze and detect. Although time constraints prevented us from implementing complete obfuscation, we explored tools such as ioccc.org, which allow the code to be rewritten in a less readable format.

Conclusion

This project of developing a companion virus was an enriching experience in the field of cybersecurity. It allowed us to understand the propagation mechanisms of viruses while respecting an educational framework. Although the virus was not intended to cause harm, the objective was to better understand the behavior of viruses and learn how to protect against them.

Reflection on Ethics and Security

Studying viruses and their propagation in an academic setting is essential to better prevent real-world attacks. The project made us aware of the risks associated with the spread of malware and the importance of securing systems and applications. As computer science students, we believe it is crucial to learn how to recognize and understand security flaws to better counter them.

Acknowledgments

I would like to sincerely thank our professors, Mr. Franco and Mr. Bouju, for their guidance and support throughout this project. A special thanks to Pierre, my teammate, for his collaboration and dedication. Finally, due to the confidential nature of the project, the source code for this application is unfortunately not available.

Introduction

Dans le cadre de notre cursus universitaire en informatique, nous avons eu l'occasion de réaliser un projet de conception d'un virus compagnon dans le cadre du cours de sécurité informatique, encadré par M. Franco et M. Bouju. Ce projet avait un objectif pédagogique spécifique : étudier et comprendre la propagation des virus, en particulier les virus compagnons, dans un environnement contrôlé. Il s'agissait principalement d'apprendre à développer la propagation d'un virus dans un contexte académique, et non de créer un virus réel destiné à nuire.

Contexte du projet

Le projet consistait à créer un virus compagnon, où le programme infecté se cache sous le nom d’un fichier légitime, trompant ainsi l'utilisateur pour qu'il exécute le programme malveillant. Nous avons été fournis avec un CD-ROM contenant des images et une application fictive appelée MediaPlayer.exe, qui était censée lire des images, mais en réalité, elle servait de vecteur pour propager le virus.

Le virus devait infecter les autres fichiers exécutables présents dans le même répertoire que le programme MediaPlayer.exe, tout en se faisant passer pour un fichier légitime afin de ne pas éveiller les soupçons de l'utilisateur. Le projet s'est inscrit dans un environnement Linux Ubuntu, et la propagation du virus s'est concentrée sur ce système d'exploitation afin de simuler un cas réel dans un cadre académique. Ce travail s'inscrivait également dans une démarche pédagogique visant à approfondir nos connaissances en sécurité informatique.

Concept de Virus Compagnon

Le virus compagnon est un type particulier de malware qui se cache sous le nom d’un programme légitime, souvent un fichier exécutable, afin de duper l’utilisateur. Il se distingue des virus classiques par sa méthode de propagation : plutôt que de s'exécuter seul, il attend qu'un fichier légitime soit exécuté par l'utilisateur pour se propager. Le programme infecté contient donc le virus, mais il se lance dans un premier temps comme s'il s'agissait du programme original, ce qui le rend difficile à détecter.

Concrètement, le virus compagnon agit comme une sorte de "parasite" en prenant le nom et le chemin d’un fichier exécuté légitime. L’utilisateur croit ainsi exécuter une application normale, alors qu'en réalité, le virus s’exécute en premier, avant de passer la main au programme légitime pour masquer sa présence et éviter de susciter des soupçons. Une fois exécuté, le virus cherche d’autres fichiers exécutables dans le répertoire cible et les infecte à leur tour, continuant ainsi sa propagation. Cette approche permet au virus compagnon de se cacher efficacement tout en continuant à se propager parmi les fichiers.

Les étapes d'infection d'un virus compagnon peuvent se résumer comme suit :

  1. Infection initiale : Lorsqu'un utilisateur exécute le programme malveillant, ce dernier renomme les fichiers exécutables présents dans le répertoire et les remplace par des copies de lui-même. Le programme légitime est ensuite exécuté.
  2. Infections ultérieures : Les fichiers infectés à leur tour deviennent des vecteurs de propagation dès qu'ils sont exécutés.

Objectifs

Les principaux objectifs du projet étaient :

  1. Développer un MediaPlayer crédible : Créer un programme fonctionnel capable de lire des images dans des formats variés (.jpg, .png, .bmp), qui servirait de vecteur pour l'exécution du virus.
  2. Créer des utilitaires : Développer plusieurs utilitaires simples destinés à être infectés par le virus. Ces utilitaires doivent interagir avec l'utilisateur et réaliser des tâches simples comme la génération de mots de passe ou la simulation de calculs.
  3. Implémenter la propagation du virus : Concevoir le mécanisme par lequel le virus se réplique et s'infecte à travers les fichiers exécutables dans le répertoire. Cela devait se faire sous la forme d'une infection initiale et d'infections ultérieures, chaque programme exécuté devenant un vecteur de propagation.

Organisation du projet

Étant donné que nous travaillions exclusivement en présentiel à l'université, nous avons choisi d'organiser le développement du projet en binôme. Cette approche nous a permis de diviser efficacement les tâches tout en travaillant en collaboration étroite pour développer le virus et les utilitaires.

  • Pierre a conçu le programme MediaPlayer.exe et l'utilitaire MonPG4.
  • Arthur a pris en charge la création des autres utilitaires (MonPG1, MonPG2, MonPG3, MonPG5).

Pour faciliter la gestion du projet, nous avons utilisé Gitlab pour la gestion de version, permettant ainsi de centraliser le code et de collaborer de manière fluide, tout en assurant une intégration continue des différentes parties du projet.

Développement des utilitaires

Les utilitaires développés remplissaient des critères spécifiques : ils devaient réaliser une tâche utile, afficher une interface graphique et interagir avec l'utilisateur. Voici les utilitaires créés :

  • MonPG1 : Un générateur de mots de passe.
  • MonPG2 : Une calculatrice basique pour les opérations simples.
  • MonPG3 : Un calculateur d'hypoténuse basé sur le théorème de Pythagore.
  • MonPG4 : Un minuteur simple avec un bouton pour démarrer/arrêter et réinitialiser.
  • MonPG5 : Un jeu du juste prix où l'utilisateur doit deviner un nombre choisi par l'ordinateur.

MediaPlayer et la propagation du virus

Le programme MediaPlayer.exe était conçu pour ressembler à une application légitime de lecture d'images. Son rôle principal était de convaincre l'utilisateur de l'exécuter. Lors de l'exécution, il effectuait les actions suivantes :

  1. Lors de la première infection, le virus renomme les exécutables existants dans le répertoire cible avec l'extension .old et crée une copie de lui-même sous les noms des fichiers originaux.
  2. Lors des infections ultérieures, chaque programme exécuté contenait une copie du virus, qui continuait à propager l'infection à d'autres fichiers exécutables du répertoire.

Les mécanismes de propagation

Le virus compagnon utilise une stratégie simple de renommage et de duplication des fichiers exécutables pour se cacher et se propager. Lors de l'infection, les étapes suivantes sont réalisées :

  1. Renommage du fichier cible avec l'extension .old.
  2. Création d'une copie du virus sous le nom du fichier d'origine, permettant au programme légitime de continuer à fonctionner sans alerter l'utilisateur.
  3. Vérification pour éviter de réinfecter les fichiers déjà compromis.

Obfuscation et sécurité

Nous avons étudié l'obfuscation du code pour rendre le virus plus difficile à analyser et à détecter. Bien que le temps ait manqué pour implémenter une obfuscation complète, nous avons exploré des outils comme ioccc.org, qui permettent de rendre le code plus opaque en le réécrivant dans un format moins lisible.

Conclusion

Ce projet de développement d'un virus compagnon a été une expérience enrichissante dans le domaine de la sécurité informatique. Il nous a permis de comprendre les mécanismes de propagation des virus tout en respectant un cadre pédagogique. Bien que le virus n'ait pas été conçu pour nuire, l'objectif était de mieux comprendre le comportement des virus et d'apprendre à s'en protéger.

Réflexion sur l’éthique et la sécurité

L'étude des virus et de leur propagation dans un cadre universitaire est essentielle pour mieux prévenir les attaques réelles. Le projet nous a permis de prendre conscience des risques associés à la propagation de malwares, et de l'importance de sécuriser les systèmes et les applications. En tant qu'étudiants en informatique, nous pensons qu'il est crucial d'apprendre à reconnaître et à comprendre les failles de sécurité pour mieux les contrer.

Remerciements

Je tenais à remercier chaleureusement nos professeurs, M. Franco et M. Bouju, pour leur accompagnement et leurs conseils tout au long de ce projet. Un grand merci également à Pierre, mon binôme, pour sa collaboration et son investissement. Enfin, en raison de la nature confidentielle du projet, le code source de cette application n'est malheureusement pas disponible.
