Project report

Eight Queens Game - Development of a Mobile Android Application

Developed by:

  • Alexy Lafosse
  • Yasin Lahrouri
  • Arthur Gillier

1. Introduction

1.1 Presentation

The Eight Queens Game project consists of implementing a mobile Android game where the user must place eight queens on an 8 × 8 chessboard without them threatening each other, following chess rules. Two queens should never share the same row, column, or diagonal.

There are 12 unique solutions to the problem, totaling 92 solutions when considering symmetrical and rotational transformations. The goal of the application is to provide an interactive and intuitive game that allows players to explore these solutions while benefiting from assistance features and customization options.

1.2 Collaborative Tool

Development was carried out collaboratively using a university GitLab repository, allowing the team to work simultaneously on different features of the project.

2. Design

2.1 Class Diagram

The application is based on an architecture with multiple classes representing different game elements (board, queens, solutions, etc.). The class diagram defines these structures and their interactions.

2.2 Use Case Diagram

A use case diagram was created to represent the different actions available to a player, including:

  • Placing a queen on the board.
  • Removing a placed queen.
  • Activating help features (highlighting attacked queens, win percentage, automatic placement).
  • Customizing the game appearance (board, pieces, background).
  • Viewing solution statistics.

3. Implementation

3.1 Storing Solutions

The 92 possible solutions are stored as arrays of size 8, where each index represents a column and each value represents a row. This method allows a complete solution to be stored in an array of size 8, reducing storage complexity.

The 12 unique solutions were manually recorded, and the remaining 80 were generated using symmetrical transformations.

3.2 Assistance Features

Three assistance features were implemented to help solve the game:

3.2.1 Highlighting Attacked Queens

  • A button allows highlighting in red the queens that are attacking each other.
  • This help feature can be activated only once per game.
  • A confirmation pop-up prevents accidental activation.

3.2.2 Displaying the Win Percentage

  • Displays in real-time the percentage of winning probability based on the current queen placement.

3.2.3 Automatic Queen Placement

  • An option allows three queens to be placed automatically based on one of the 92 correct solutions.

3.3 Removal Features

  • A "Reset" button allows removing all placed queens from the board, making it easy to restart quickly.

3.4 Confirmation Pop-ups

Two types of pop-ups confirm irreversible actions:

3.4.1 Standard Pop-up

  • Used to confirm the final validation of the game.

3.4.2 Dismissible Pop-up

  • Allows the user to disable the pop-up permanently for certain confirmation messages.

4. Aesthetic Features and Customization

The application offers several options to enhance the user experience and customize the game interface:

4.1 Backgrounds and Themes

  • 4 different backgrounds (one created by an independent artist).
  • 4 board color schemes.
  • 4 different queen styles.

4.2 Animations and Interactions

  • Button press effect to enhance interaction.
  • Smooth animations improving the game's overall aesthetics.

4.3 Music and Sound Effects

  • Integration of royalty-free background music using Android's MediaPlayer class.
  • Option to enable or disable the music at any time.

5. Technical Details

5.1 Rendering and Storage Management

  • The graphical interfaces were coded in XML (without any specific framework).
  • Displaying the chessboard and queens required extensive work on graphical rendering.

5.2 Software Structure

  • The Material & Interface CosmeticListener class enables dynamic UI updates based on the player's customization choices.
  • The BoutonListener class manages pop-up display and button animations.

6. Work Distribution

Member Responsibilities
Arthur Gillier Responsible for graphical interface and game customization (backgrounds, colors, queen styles).
Alexy Lafosse Development of core game features (logic, placement, help features). Management of Git repository and merges.
Yasin Lahrouri Design and integration of UI/UX elements (buttons, logo, animations). Music integration.

7. Conclusion

This project allowed us to develop various skills, including:

  • Development of an Android application in Java
  • Collaborative work using GitLab
  • Conflict management and Git branch merging
  • Experience in UI/UX design and interface customization
  • Deepening knowledge in data management and game algorithms

We faced several challenges, particularly handling graphical rendering and integrating interactive animations. Despite this, the application is functional and provides a smooth and customizable user experience.

8. Future Improvements

Although the application is operational, several improvements could be considered:

  • Adding a multiplayer mode allowing two players to compete.
  • Recording scores and played games.
  • Implementing an interactive tutorial for new players.
  • Enhancing graphical performance using an optimized rendering engine.

Jeu des 8 Dames - Développement d'une Application Mobile Android

Réalisé par :

  • Alexy Lafosse
  • Yasin Lahrouri
  • Arthur Gillier

1. Introduction

1.1 Présentation

Le projet Jeu des 8 dames consiste en l’implémentation d’un jeu mobile Android où l’utilisateur doit placer huit dames sur un échiquier de 8 × 8 cases sans que celles-ci ne se menacent mutuellement, selon les règles du jeu d’échecs. Deux dames ne doivent jamais partager la même rangée, colonne ou diagonale.

Il existe 12 solutions uniques au problème, qui donnent un total de 92 solutions si l'on prend en compte les symétries et rotations. L'objectif de l'application est d'offrir un jeu interactif et intuitif permettant aux joueurs d'explorer ces solutions tout en bénéficiant d'aides et de personnalisations.

1.2 Outil collaboratif

Le développement a été réalisé en collaboration sur un dépôt GitLab universitaire permettant à l’équipe de travailler simultanément sur différentes fonctionnalités du projet.

2. Conception

2.1 Diagramme de classe

L’application repose sur une architecture en plusieurs classes permettant de modéliser les différents éléments du jeu (plateau, dames, solutions, etc.). Le diagramme de classe définit ces structures et leurs interactions.

2.2 Diagramme des cas d’utilisation

Un diagramme des cas d’utilisation a été réalisé pour représenter les différentes actions possibles par un joueur, notamment :

  • Placer une dame sur l’échiquier.
  • Retirer une dame placée.
  • Activer des aides (affichage des dames attaquées, pourcentage de victoire, placement automatique).
  • Personnaliser l’apparence du jeu (plateau, pièces, arrière-plan).
  • Consulter les statistiques des solutions.

3. Implémentation

3.1 Stockage des solutions

Les 92 solutions possibles sont stockées sous forme de tableaux d’entiers de taille 8, où chaque index représente une colonne et chaque valeur représente une ligne. Cette approche permet de stocker une solution complète en un tableau de taille 8, réduisant ainsi la complexité de stockage.

Les 12 solutions uniques ont été enregistrées manuellement, puis les 80 autres solutions ont été générées en appliquant des transformations symétriques.

3.2 Fonctionnalités d’aides

Trois fonctionnalités d’aides ont été mises en place pour faciliter la résolution du jeu :

3.2.1 Affichage des dames attaquées

  • Un bouton permet d’afficher en rouge les dames qui s’attaquent mutuellement.
  • Cette aide ne peut être activée qu’une seule fois par partie.
  • Une confirmation via pop-up est nécessaire pour éviter une activation accidentelle.

3.2.2 Affichage du pourcentage de victoire

  • Affichage en temps réel du pourcentage de chances de victoire en fonction de la disposition actuelle des dames.

3.2.3 Placement automatique des dames

  • Une option permet de placer trois dames automatiquement selon une solution correcte parmi les 92 possibles.

3.3 Fonctionnalités de suppression

  • Un bouton "Réinitialiser" permet de supprimer toutes les dames placées sur l’échiquier, facilitant ainsi un redémarrage rapide.

3.4 Pop-up de confirmation

Deux types de pop-ups sont utilisés pour confirmer certaines actions irréversibles :

3.4.1 Pop-up classique

  • Utilisé pour confirmer la validation finale de la partie.

3.4.2 Pop-up retirable

  • Permet à l’utilisateur de désactiver définitivement l’affichage de certains messages de confirmation.

4. Fonctionnalités esthétiques et personnalisation

L'application propose plusieurs options pour améliorer l'expérience utilisateur et personnaliser l’interface du jeu :

4.1 Arrière-plans et thèmes

  • 4 fonds d’écran différents (dont un réalisé par un artiste indépendant).
  • 4 styles de couleurs du plateau.
  • 4 styles de dames différentes.

4.2 Animations et interactions

  • Effet d’écrasement des boutons lors d’un clic pour renforcer l’interaction.
  • Animations fluides améliorant l’esthétique générale du jeu.

4.3 Musique et sonorisation

  • Intégration d’une musique de fond libre de droits via la classe MediaPlayer d’Android.
  • Possibilité d’activer ou de désactiver la musique à tout moment.

5. Détails Techniques

5.1 Gestion du rendu et du stockage

  • Les interfaces graphiques ont été codées en XML (sans framework spécifique).
  • L’affichage de l’échiquier et des dames a nécessité un travail approfondi sur le rendu graphique.

5.2 Structure logicielle

  • La classe Material & Interface CosmeticListener permet la mise à jour dynamique de l’affichage en fonction des choix de personnalisation du joueur.
  • La classe BoutonListener gère l’affichage des pop-ups et l’animation des boutons.

6. Répartition du Travail

Membre Responsabilités
Arthur Gillier Responsable de l’interface graphique et de la personnalisation du jeu (fonds d’écran, couleurs, styles de dames).
Alexy Lafosse Développement des fonctionnalités principales (logique du jeu, placements, aides). Gestion du dépôt Git et des fusions.
Yasin Lahrouri Conception et intégration des éléments UI/UX (boutons, logo, animations). Intégration de la musique.

7. Conclusion

Ce projet nous a permis de développer de nombreuses compétences, notamment :

  • Développement d’une application Android en Java
  • Travail collaboratif avec GitLab
  • Gestion des conflits et fusion des branches Git
  • Expérience en UI/UX design et personnalisation d’interfaces
  • Approfondissement de la gestion des données et des algorithmes de jeu

Nous avons dû relever plusieurs défis, notamment la gestion du rendu graphique et l’intégration d’animations interactives. Malgré cela, l’application est fonctionnelle et offre une expérience utilisateur fluide et personnalisable.

8. Perspectives d’amélioration

Bien que l’application soit opérationnelle, plusieurs améliorations pourraient être envisagées :

  • Ajout d’un mode multijoueur permettant à deux joueurs de s’affronter.
  • Enregistrement des scores et des parties jouées.
  • Mise en place d’un tutoriel interactif pour les nouveaux joueurs.
  • Amélioration des performances graphiques via un moteur de rendu optimisé.
